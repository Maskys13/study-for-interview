### React虚拟DOM的工作原理
```
1.虚拟DOM的实现。虚拟DOM本质上是JavaScript对象，用于表示真实DOM的结构和状态。在React中，所有的DOM操作都是通过虚拟DOM进行的。当数据发生变化时，React会重新构建整个虚拟DOM树。12345
2.虚拟DOM与真实DOM的对比。当状态变更时，React会对比新旧虚拟DOM树之间的差异，这个过程称为“diff”。通过这种方式，React可以找出需要更新的部分，并生成一个patch对象。1356
更新过程。根据生成的patch对象，React会对真实DOM进行最小化更新。这种更新方式可以避免不必要的DOM操作，从而提高性能。156
3.批量处理更新。React能够批量处理虚拟DOM的刷新。如果两次数据变化被合并，例如先改变节点内容，然后又改回原状，React会认为UI没有发生变化，从而避免进行不必要的DOM更新。1
性能优化。React的diff算法是虚拟DOM的核心，它通过减少比较步骤来优化性能。传统的diff算法复杂度为O(n^3)，而React的diff算法通过优化，提高了效率。
```
### React 虚拟DOM 对性能方面有什么帮助？
```
1. 减少回流与重绘，通过算法优化将多个连续的操作合并成一次更新。
2. 批量更新，当多个状态发生变化时，合并为一次更新操作，只对虚拟DOM进行一次和真实DOM的比较操作，避免了不必要的中间过程。
3. 最小化真是DOM操作，通过虚拟DOM，先把对真实DOM的操作应用到虚拟DOM上然后通过与上一次状态的真实DOM进行比较，减少对真实DOM的频繁操作。
```

### React中的错误边界
```
Error boundaries（错误边界）是一个React组件，它可以捕获并打印发生在其子组件数任何位置的JS错误，然后，渲染出备用UI，而非崩溃了的子组件。错误边界可以在渲染期间、生命周期方法和整个组件树的构造函数中捕获错误。
错误边界无法捕获以下几种错误：
事件处理
异步代码
服务端渲染
自身错误
只要在Class组件中定义static getDerivedStateFromError或componentDidCatch这两个生命周期方法中的任意一个/两个，那么这个组件就是一个错误边界组件了。
```
